# Plan: Backend Propio para Fotos Jacky Ecommerce

## Contexto y MotivaciГіn

Tienda Nube cobra $80.000 ARS/mes por alojar la tienda. El objetivo es construir un
**backend propio y autoalojado** que replique (y supere) esas capacidades, manteniendo
el frontend premium ya desarrollado (JackyV2) y conectГЎndolo a infraestructura gratuita
o de costo mГӯnimo.

El stack estГЎ elegido para que **vos puedas administrarlo** con mi ayuda en el futuro,

## MГіdulos del Backend вҖ” Detalle Completo

### MГіdulo 1: Productos y CategorГӯas

**Entidades en la base de datos:**
- `categories` вҖ” Cuadros, Fotolibros, Impresiones, Regalos
- `products` вҖ” Con tГӯtulo, descripciГіn, precio, SKU, stock, imГЎgenes
- `product_variants` вҖ” TamaГұos (10x15, A4, A3, etc.), acabados (mate/brillo), etc.
- `product_images` вҖ” MГәltiples fotos por producto con orden

**Endpoints:**
```
GET  /api/products              вҶ’ Listar productos (con filtros y paginaciГіn)
GET  /api/products/{slug}       вҶ’ Detalle de un producto
GET  /api/categories            вҶ’ Listar categorГӯas
GET  /api/categories/{slug}     вҶ’ Productos por categorГӯa
POST /api/admin/products        вҶ’ Crear producto (auth requerida)
PUT  /api/admin/products/{id}   вҶ’ Editar producto
DEL  /api/admin/products/{id}   вҶ’ Eliminar producto
```

---

### MГіdulo 2: Carrito de Compras

**Funcionamiento:** El carrito se guarda en `localStorage` del browser. No requiere login.
Antes de hacer checkout, se valida stock en el servidor.

**Endpoints:**
```
POST /api/cart/validate          вҶ’ Valida items del carrito (stock, precios)
```

---

### MГіdulo 3: Clientes y Г“rdenes

**Entidades:**
- `customers` вҖ” Email, nombre, DNI, telГ©fono, direcciГіn
- `orders` вҖ” Estado, items, subtotal, envГӯo, total, payment_id de MP
- `order_items` вҖ” Producto, variante, cantidad, precio unitario
- `order_status_history` вҖ” Trazabilidad de cambios de estado

**Estados de una orden:**
```

                      вҶҳ cancelled / refunded
```

**Endpoints:**
```
POST /api/checkout               вҶ’ Crea orden + inicia pago en MP
GET  /api/orders/{order_id}      вҶ’ Estado de una orden (pГәblica, con token)
GET  /api/admin/orders           вҶ’ Listar todas las Гіrdenes
PUT  /api/admin/orders/{id}      вҶ’ Actualizar estado, cargar nГәmero de envГӯo
```

### MГіdulo 4: Pagos con Mercado Pago

**Flujo completo:**

```
1. Cliente llena datos вҶ’ POST /api/checkout
2. Backend crea preferencia en MP con:
   - Items del carrito
   - Datos del cliente
   - URLs de retorno (Г©xito/error/pendiente)
   - Webhook URL para notificaciones
3. Backend devuelve init_point (URL de pago de MP)
4. Frontend redirige al checkout de MP
5. Cliente paga (tarjeta, dГ©bito, transferencia, efectivo)
6. MP notifica al webhook: POST /api/webhooks/mercadopago
7. Backend confirma pago, actualiza estado de orden
8. Se envГӯa email de confirmaciГіn al cliente
```

**MГ©todos de pago soportados automГЎticamente por MP:**
- рҹ’і Tarjetas de crГ©dito (Visa, Mastercard, Amex, Naranja, etc.)
- рҹҸҰ Tarjetas de dГ©bito
- рҹ“ұ Transferencia bancaria / CVU (Alias/CBU)
- рҹ’ө Efectivo (Rapipago, PagoFГЎcil)
- рҹ“І Mercado CrГ©dito (cuotas sin tarjeta)

**ComisiГіn MP:** 5,99% por venta (sin comisiГіn mensual fija).

---

### MГіdulo 5: EnvГӯos

**Estrategia:** Calculamos cotizaciГіn en tiempo real antes del pago, con mГәltiples opciones.

#### Couriers integrados:

**a) Andreani (recomendado principal)**
- API REST moderna, documentaciГіn clara
- Cobertura nacional
- Retiro en sucursal o domicilio
- Tracking incluido
- Requiere cuenta de empresa (gratuito)

**b) OCA e-Pak**
- API disponible
- Muy usado para ecommerce en Argentina
- TambiГ©n ofrece retiro en sucursal

**c) Correo Argentino**
- MГЎs econГіmico para paquetes chicos
- Cobertura total del paГӯs
- API disponible vГӯa integradores

**Flujo de cotizaciГіn:**
```
1. Cliente ingresa cГіdigo postal de destino
2. POST /api/shipping/quote con: [CP destino, peso, dimensiones]
3. Backend consulta a Andreani + OCA en paralelo
4. Devuelve array de opciones con precio y plazo estimado
5. Cliente elige opciГіn вҶ’ queda guardada en la orden
```

**DespuГ©s del envГӯo (admin):**
```
PUT /api/admin/orders/{id}/shipping вҶ’ Carga nГәmero de tracking
вҶ’ Sistema notifica al cliente por email con link de seguimiento
```

## Base de Datos вҖ” Esquema SQL

```sql
-- CategorГӯas
CREATE TABLE categories (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    name VARCHAR(100) NOT NULL,
    slug VARCHAR(100) UNIQUE NOT NULL,
    description TEXT,
    image_url TEXT,
    position INT DEFAULT 0,
    active BOOLEAN DEFAULT true,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Productos
CREATE TABLE products (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    category_id UUID REFERENCES categories(id),
    name VARCHAR(200) NOT NULL,
    slug VARCHAR(200) UNIQUE NOT NULL,
    description TEXT,
    base_price DECIMAL(10,2) NOT NULL,
    sku VARCHAR(100),
    weight_grams INT DEFAULT 500,
    dimensions_cm JSONB,           -- {"l":30,"w":20,"h":2}
    active BOOLEAN DEFAULT true,
    featured BOOLEAN DEFAULT false,
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Variantes (tamaГұos, acabados)
CREATE TABLE product_variants (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    product_id UUID REFERENCES products(id),
    name VARCHAR(100) NOT NULL,    -- "A4 - Mate"
    sku VARCHAR(100),
    price_modifier DECIMAL(10,2) DEFAULT 0,
    stock INT DEFAULT 0,
    active BOOLEAN DEFAULT true
);

-- ImГЎgenes de productos
CREATE TABLE product_images (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    product_id UUID REFERENCES products(id),
    url TEXT NOT NULL,
    alt TEXT,
    position INT DEFAULT 0
);

-- Clientes
CREATE TABLE customers (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(200) UNIQUE NOT NULL,
    first_name VARCHAR(100),
    last_name VARCHAR(100),
    phone VARCHAR(50),
    dni VARCHAR(20),
    created_at TIMESTAMPTZ DEFAULT NOW()
);

-- Direcciones de clientes
CREATE TABLE addresses (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    customer_id UUID REFERENCES customers(id),
    street VARCHAR(200),
    street_number VARCHAR(20),
    floor_apt VARCHAR(50),
    city VARCHAR(100),
    province VARCHAR(100),
    postal_code VARCHAR(10),
    is_default BOOLEAN DEFAULT false
);

-- Г“rdenes
CREATE TABLE orders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_number SERIAL,           -- NГәmero legible: #0001, #0002...
    customer_id UUID REFERENCES customers(id),
    status VARCHAR(50) DEFAULT 'pending_payment',
    subtotal DECIMAL(10,2),
    shipping_cost DECIMAL(10,2),
    total DECIMAL(10,2),
    shipping_address JSONB,
    shipping_method VARCHAR(100),  -- "andreani_domicilio"
    shipping_tracking VARCHAR(200),
    mp_preference_id VARCHAR(200),
    mp_payment_id VARCHAR(200),
    mp_status VARCHAR(50),
    notes TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);

-- Items de Гіrdenes
CREATE TABLE order_items (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_id UUID REFERENCES orders(id),
    product_id UUID REFERENCES products(id),
    variant_id UUID REFERENCES product_variants(id),
    product_name VARCHAR(200),
    variant_name VARCHAR(100),
    quantity INT NOT NULL,
    unit_price DECIMAL(10,2) NOT NULL
);

-- Historial de estados
CREATE TABLE order_status_history (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    order_id UUID REFERENCES orders(id),
    status VARCHAR(50),
    note TEXT,
    created_at TIMESTAMPTZ DEFAULT NOW()
);
```
